# venus — review conventions

Rules below were extracted from real review feedback in level-connections/venus (generated by RepoMind; evidence links point to the original discussions).

## Security

- Authorize access to user-specific data using dedicated policy classes and pass the specific model instance to authorization methods. _(seen in 2 PR(s), 1 reviewer(s) — [example](https://github.com/level-connections/venus/pull/567#discussion_r2487968192))_
- Place authorization checks early in policy methods and use the most restrictive check first. _(seen in 2 PR(s), 1 reviewer(s) — [example](https://github.com/level-connections/venus/pull/608#discussion_r2545121069))_
- Use the framework's validation system instead of manual checks for input validation. _(seen in 1 PR(s), 1 reviewer(s) — [example](https://github.com/level-connections/venus/pull/808#discussion_r2981208773))_
- Avoid exposing sensitive terms in public URL paths. _(seen in 1 PR(s), 1 reviewer(s) — [example](https://github.com/level-connections/venus/pull/1020#discussion_r3380960385))_
- Conditionally render UI elements based on feature flags or user settings to prevent unauthorized information exposure. _(seen in 1 PR(s), 1 reviewer(s) — [example](https://github.com/level-connections/venus/pull/1020#discussion_r3380889169))_
- Restrict user navigation during critical flows to prevent data integrity issues. _(seen in 1 PR(s), 1 reviewer(s) — [example](https://github.com/level-connections/venus/pull/808#discussion_r2999577630))_
- Verify that authorization logic matches the intended user story before applying it. _(seen in 1 PR(s), 1 reviewer(s) — [example](https://github.com/level-connections/venus/pull/426#discussion_r2290111016))_
- When comparing database identifiers across different models, ensure the comparison involves the same conceptual entity. _(seen in 1 PR(s), 1 reviewer(s) — [example](https://github.com/level-connections/venus/pull/608#discussion_r2544772528))_
